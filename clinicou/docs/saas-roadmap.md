# SaaS Roadmap

## Phase 1 - Platform Foundation

- Keep schema changes in Supabase migrations generated by the CLI.
- Maintain `dev`, `staging` and `prod` Supabase projects.
- Run `npm run verify` in CI before deploy.
- Run Supabase advisors before promoting schema changes.

## Phase 2 - Billing, Plans and Onboarding

Implemented foundation:

- `billing_plans`
- `clinic_subscriptions`
- `onboarding_tasks`

Next implementation steps:

- Wire checkout creation in an Edge Function.
- Verify billing webhooks and update `clinic_subscriptions`.
- Gate limits from `billing_plans.limits`.
- Render onboarding tasks in Admin and mark tasks complete as setup happens.

## Phase 3 - Integrations and Scale

Implemented foundation:

- `integration_connections`
- `automation_rules`
- `document_templates`
- `analytics_events`
- Edge Function contracts for billing, WhatsApp, document rendering and analytics ingest.

Next implementation steps:

- Connect Meta WhatsApp API in `whatsapp-dispatch`.
- Render PDFs and upload to private Supabase Storage in `document-render`.
- Insert validated product events in `analytics-ingest`.
- Add background automation processing for reminders and post-visit messages.
- Add query/performance tests for high-volume agenda and patient search.
