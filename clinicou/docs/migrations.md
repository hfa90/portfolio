# Migrations

The current baseline schema is in `supabase/schema.sql`.

The Supabase CLI was not available in this workspace when this file was created, so no timestamped migration file was generated by hand. Generate the first migration with the CLI:

```bash
supabase migration new baseline_schema
```

Then copy the contents of `supabase/schema.sql` into the generated SQL file.

For future schema changes:

```bash
supabase migration new short_description
supabase db push
supabase db advisors
```

Do not manually invent migration filenames. Let the Supabase CLI create them.
